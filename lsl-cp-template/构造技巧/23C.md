# 23C

https://codeforces.com/contest/23/problem/C

这类题目通常有一个共同的底层模型：
> 假设有 $2k+1$ 个物品，每个物品有两个维度的属性 $A$ 和 $B$（通常要求是非负数）。你需要选出 $k+1$ 个物品，使得选出物品的属性 $A$ 总和 $\ge$ 所有物品 $A$ 总和的一半，**同时**属性 $B$ 的总和也 $\ge$ 所有物品 $B$ 总和的一半。

直接同时兼顾两个条件非常困难，但这种构造法实现了“降维打击”。以下是它背后的核心数学原理解析：

### 原理一：利用“排序 + 错位配对”让属性 A 无条件达标

假设我们已经按照属性 $A$ 对所有物品进行了**升序排序**：
$A_1 \le A_2 \le A_3 \le \dots \le A_{2k} \le A_{2k+1}$

现在来看你提到的两种构造方案：
* **方案一（奇数下标）：** $S_1 = \{1, 3, 5, \dots, 2k-1, 2k+1\}$
* **方案二（偶数下标 + 最后一个）：** $S_2 = \{2, 4, 6, \dots, 2k, 2k+1\}$

为什么这两种方案都能保证属性 $A$ 的和不小于总和的一半呢？这就是**相邻配对（交替支配）**的奇妙之处：

**对于方案一（奇数下标）：**
除了 $A_1$ 之外，我们将后面的元素两两分组比较：$(A_2, A_3), (A_4, A_5), \dots, (A_{2k}, A_{2k+1})$。
因为是升序排列，必然有 $A_3 \ge A_2$、$A_5 \ge A_4$ …… $A_{2k+1} \ge A_{2k}$。
将它们相加可得：$A_3 + A_5 + \dots + A_{2k+1} \ge A_2 + A_4 + \dots + A_{2k}$。
左边再加上一个 $A_1$（只要 $A_1 \ge 0$），这就严格证明了：**方案一选出的 $A$ 之和 $\ge$ 剩下未选的 $A$ 之和**。既然大于等于另一半，它必然大于等于总和的一半。

**对于方案二（偶数下标 + 最后一个）：**
我们从头开始两两分组比较：$(A_1, A_2), (A_3, A_4), \dots, (A_{2k-1}, A_{2k})$。
因为升序，必然有 $A_2 \ge A_1$、$A_4 \ge A_3$ …… $A_{2k} \ge A_{2k-1}$。
相加可得：$A_2 + A_4 + \dots + A_{2k} \ge A_1 + A_3 + \dots + A_{2k-1}$。
左边再加上强制选中的最后一个元素 $A_{2k+1}$，同样证明了：**方案二选出的 $A$ 之和 $\ge$ 剩下未选的 $A$ 之和**。因此方案二的 $A$ 也绝对达标。

---

### 原理二：利用“交并集”触发抽屉原理让属性 B 达标

既然两种方案在属性 $A$ 上都是合法的“备胎”，我们该如何保证属性 $B$ 也达标呢？

仔细观察 $S_1$ 和 $S_2$ 这两个下标集合：
* $S_1$ 包含了所有的奇数下标。
* $S_2$ 包含了所有的偶数下标，以及最后一个元素。

你会发现，$S_1$ 和 $S_2$ 的**并集是所有的元素**，而它们的**交集只有最后一个元素**（下标 $2k+1$ 既是奇数，也在方案二中被明确加入）。

这就意味着，如果我们把方案一在属性 $B$ 上的总和，加上方案二在属性 $B$ 上的总和，结果正好等于：
$$\sum_{i \in S_1} B_i + \sum_{i \in S_2} B_i = \sum_{i=1}^{2k+1} B_i + B_{2k+1}$$
（即：所有物品的 $B$ 总和，加上被重复计算了一次的 $B_{2k+1}$）

只要 $B_{2k+1} \ge 0$，就必然有：
$$\sum_{i \in S_1} B_i + \sum_{i \in S_2} B_i \ge \text{所有物品 } B \text{ 的总和}$$

**最核心的推论来了（抽屉原理 / 平均值原理）：**
既然这两个方案的属性 $B$ 综合加起来大于等于一个总和（100%），那么**这两个方案中，必定至少有一个方案的属性 $B$ 总和 $\ge$ 50%（总和的一半）！**

---

### 总结

这种精妙构造的思维本质可以归纳为两步：
1.  **解耦与兜底：** 通过在一维上排序并利用“奇偶交替”，我们构造出了两套方案，这两套方案都能单方面、无条件地克服条件一（属性 $A$）的限制。
2.  **必然性覆盖：** 这两套方案巧妙地互补，拼凑出了整个集合（甚至在末尾还有冗余），从而保证了它们在应对条件二（属性 $B$）时，绝对不可能同时拉胯。

只要你写代码时分别计算一下方案一和方案二，看看哪个方案的属性 $B$ 更符合要求，选那个输出，就一定能同时满足题目给出的双重限制。下次再遇到需要“在多个条件中同时拿取一半”的博弈或分配问题，你就可以直接往**“一维排序锁定单调性，奇偶划分双线兜底”**的方向去思考了！

这种**先利用排序/结构强制满足一个维度，再利用抽屉原理/奇偶性在另一个维度上‘二选一’保底**的思维，在算法竞赛（尤其是 Codeforces 的 Constructive Algorithms 构造题）中绝对算得上是核心套路之一。

这种思想的本质是**降维与解耦**。除了你刚才遇到的“双重条件 $\ge \frac{1}{2}$”问题，还有几个非常经典的 Trick 也使用了极其类似的底层逻辑：

### 1. 树的黑白染色选大头（图论中的抽屉原理）
**常见题型：** 给定一棵 $N$ 个节点的树，要求你选出一个大小至少为 $\lceil N/2 \rceil$ 的独立集（选出的点之间两两没有边相连）。

**神仙 Trick：** 直接对这棵树进行**黑白染色（二分图染色）**。
随便选一个点作为根，它染成黑色，它的儿子染成白色，孙子再染成黑色……这样交替下去。 
因为树天然没有任何奇环，所以它**一定是一个完美的二分图**，同色节点之间绝对没有边相连。

**如何保底？**
黑色节点总数 + 白色节点总数 $= N$。
根据抽屉原理，**必然至少有一种颜色的节点数量 $\ge \lceil N/2 \rceil$**。
你只需要统计一下哪种颜色的点多，直接把那个颜色的所有点输出即可！完美满足“独立集”和“数量达标”两个要求。

### 2. 相邻配对抵消法（数组差值控制）
**常见题型：** 给定 $2n$ 个正整数，要求将它们分成大小相等的两个集合 $A$ 和 $B$（各 $n$ 个元素），使得它们的所有元素之和的差值，不能超过这批数字中的最大值与最小值之差。

**神仙 Trick：**
先对数组从小到大排序：$x_1 \le x_2 \le x_3 \le \dots \le x_{2n}$。
然后我们**把相邻的元素两两绑定成一对**：$(x_1, x_2), (x_3, x_4), \dots, (x_{2n-1}, x_{2n})$。

**如何保底？**
对于每一对 $(x_{2i-1}, x_{2i})$，我们把大的（偶数下标）放进集合 $A$，小的（奇数下标）放进集合 $B$。
集合 $A$ 和 $B$ 的差值就是每一对的差值之和：
$\Delta = (x_2 - x_1) + (x_4 - x_3) + \dots + (x_{2n} - x_{2n-1})$

因为数组是有序的，你可以发现这个式子在数学上被极其巧妙地放缩了：
$\Delta \le x_2 + (x_4 - x_3) + \dots + (x_{2n} - x_{2n-1}) - x_1$
由于 $x_2 \le x_3, x_4 \le x_5$，中间的项相互抵消（放缩），最终整个差值 $\Delta$ 绝对会被死死地压在 $x_{2n} - x_1$ （即最大值减最小值）以内。通过排序+相邻配对，极其棘手的差值控制问题迎刃而解。

### 3. 跨越半个数组的错位匹配（非同类匹配问题）
**常见题型：** 给定 $N$ 个物品（$N$ 为偶数），每个物品有一个种类。已知没有任何一个种类的物品数量超过 $N/2$。要求你把这 $N$ 个物品两两配对，使得**每一对物品的种类都不一样**。

**神仙 Trick：**
不要去用贪心或者优先队列搞什么“每次拿剩下最多的两个配对”，那种做法写起来极其繁琐且容易 WA。
正确的姿势是：**把所有物品按种类排序**（把相同种类的物品全挤在一起），排成一个长度为 $N$ 的数组 $Arr$。
然后，直接让第 $i$ 个物品，和第 $i + N/2$ 个物品配对！

**如何保底？**
因为没有任何一种物品的数量超过 $N/2$（这是题目的保底条件），所以 $Arr[i]$ 和 $Arr[i + N/2]$ 这两个位置之间的距离刚好跨越了半个数组。它们**绝对不可能是同一种物品**。
只需两行代码（排序 + 错位遍历），就完美解决了看似需要复杂图论匹配的问题。

---

### 总结这些 Trick 的共性

仔细观察这些题目，就会发现，这些题目都有一个共同的破题公式：
**无序变有序（或结构化） $\rightarrow$ 建立绑定关系（奇偶/相邻/平移） $\rightarrow$ 触发数学上的必然定理（抽屉原理/放缩/鸽巢）**

当你再遇到那种“条件看起来很苛刻，要求你构造一个合法方案”的题目时，如果正面贪心行不通，不妨试着先给数据**排个序**或者**分个奇偶**，然后大胆地去猜想：是不是只要闭着眼睛分成两拨，比较一下，就必然有一个是答案？这种“降维打击”是 CF 题中最迷人的部分！
